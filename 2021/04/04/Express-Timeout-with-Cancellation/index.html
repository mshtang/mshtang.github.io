It’s possible to use Task.WhenAny to implement timeouts (e.g. using Task.Delay as one of the tasks), but it’s not recommended. Cancellation is a natural solution for timeout situations, because a timeout is just one type of cancellation request. The code that needs to be canceled merely observe the cancellation token, it should neither know nor care that the cancellation source is a timer.
